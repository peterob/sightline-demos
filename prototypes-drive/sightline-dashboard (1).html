Active Policies

POL-001 Active

Spending Authorization Limits

IF amount ≤ $5,000
THEN Department Manager approval required
IF $5,000 < amount ≤ $25,000
THEN VP + Finance Director approval required
IF amount > $25,000
THEN CFO approval required + Board notification
Effective: Jan 1, 2025 Owner: Finance
POL-002 Active

Budget Variance Alerts

IF committed > 80% of budgeted
THEN Alert Department Manager + Finance
IF committed > 95% of budgeted
THEN Freeze new POs pending CFO review
IF actual > budgeted (overage)
THEN Require variance explanation + reforecast
Effective: Jan 1, 2025 Owner: Finance
POL-003 Active

Vendor Payment Terms

IF vendor.tier = "Strategic"
THEN Net 30 terms, auto-pay enabled
IF vendor.tier = "Standard"
THEN Net 45 terms, manual approval batch
IF invoice.early_pay_discount ≥ 2%
THEN Flag for early payment consideration
Effective: Jan 1, 2025 Owner: AP/Treasury
POL-004 Active

Three-Way Match Validation

IF invoice.amount = PO.amount ± 2%
THEN Auto-match, proceed to payment queue
IF variance > 2% AND ≤ 10%
THEN Route to Dept Manager for approval
IF variance > 10% OR missing receipt
THEN Hold payment, create exception ticket
Effective: Jan 1, 2025 Owner: AP/Procurement
POL-005 Active

Capital Expenditure Classification

IF asset.useful_life > 1 year AND amount > $5,000
THEN Classify as CapEx, add to fixed asset register
IF category = "Software" AND term > 12 months
THEN Evaluate ASC 842 lease treatment
IF CapEx.total_ytd > board_approved_budget
THEN Require board approval for incremental spend
Effective: Jan 1, 2025 Owner: Controller
POL-006 Active

Commitment Lifecycle State Machine

STATE Planned → Approved
REQUIRES budget_available = true, approver.sign_off
STATE Approved → Committed
REQUIRES PO.issued = true, vendor.confirmed
STATE Committed → Settled
REQUIRES three_way_match = pass, payment.confirmed
Effective: Jan 1, 2025 Owner: Finance Ops
