State Machine

Valid Transitions

COMMITTED → FUNDED
FUNDED → DISPUTE_WINDOW
DISPUTE_WINDOW → SETTLED (auto, 72h)
DISPUTE_WINDOW → DISPUTED
DISPUTED → SETTLED | REFUNDED

Invariants

  • ✓ State can only move forward (no rollback)
  • ✓ Every transition requires a signed event
  • ✓ Hash chain is append-only, immutable
  • ✓ Funds release only from terminal states
  • ✓ Any party can recompute state from events

Hash Chain Verification

// Anyone can verify the chain independently
function verifyChain(events) {
  let priorHash = "GENESIS";
  
  for (const event of events) {
    const computed = sha256(
      event.event_id + "|" +
      event.deal_id + "|" +
      event.event_type + "|" +
      event.actor + "|" +
      event.timestamp + "|" +
      JSON.stringify(event.data) + "|" +
      priorHash
    );
    
    if (computed !== event.hash) {
      throw new Error(`Tampering detected at ${event.event_id}`);
    }
    
    priorHash = event.hash;
  }
  
  return { verified: true, length: events.length };
}

Why Not Blockchain?

We don't need:
  • • Decentralized consensus (parties are known)
  • • Token economics (using real USD)
  • • Global state (just our deals)
  • • Permissionless access (invited parties only)
We do need:
  • • Immutability (hash chains provide this)
  • • Multi-party visibility (shared ledger)
  • • Verifiability (anyone can check the math)
  • • Speed (instant, not 12-second blocks)
